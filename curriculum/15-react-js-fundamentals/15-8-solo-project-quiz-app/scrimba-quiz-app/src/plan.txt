A quiz app that fetches five questions from a trivia API, let's the user answer, then checks answers, shows score and shows correct/incorrect answers

User Happy Path
1. User lands on start screen
2. User clicks start btn
3. App loads questions (perhaps need loading spinner)
4. User selects answers for each questions
5. User clicks Check Answers button
6. App checks score, shows score and highlights correct/incorrect answers
7. User clicks Play again btn

Rough Data model
Once the user clicks Start Quiz btn, the app fetches the questions and correct/incorrect answers from an API. The app then needs to display those questions and answers in QA components. 
The app needs to store the questions and answers in state. It needs to store selected answers in state and selected answers need to be blue. Once the the user checks answers, the app needs to compare selected answers to correct answers and mark the test and update the style of correct/incorrect answers to green/red.
State
> original questions and answers 
> selected answers (could possibly use derived state for this and update isSelected on the original state)

Core data model (state shape)
What must the app remember?
gameStarted
    > false: show start page
    > true: set QA state by fetching from API and render the QA components
        >> this will need a useEffect so that the API call is not run every render
userFinished
    > false: check answer button displayed, selected answers are blue, user can still change answers
    > true: answers need to be checked and score calculated and displayed, correct/incorrect highlighted, play again button needs to be displayed
    |
    >> play again btn shouldn't take user back to start quiz page, it should just fetch new QAs and reset state
